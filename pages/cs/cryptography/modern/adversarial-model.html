Adversarial Model

In modern cryptography, security is defined relative to an adversarial model: a formal specification of the adversary’s capabilities, resources, and goals. This ensures that security claims are precise, testable, and mathematically rigorous rather than heuristic.

Definition of an Adversary

An adversary is an algorithm, typically modeled as a probabilistic polynomial-time (PPT) entity, which attempts to compromise one or more security properties of a cryptographic scheme. Examples of properties include:

  • Confidentiality: The adversary cannot learn any information about plaintexts from ciphertexts.
  • Integrity: The adversary cannot forge valid messages or modify them undetectably.
  • Authentication: The adversary cannot impersonate legitimate parties.

Types of Adversaries

  • Passive Adversary: Observes communications but does not alter them.
  • Active Adversary: Can inject, modify, or delete messages.
  • Adaptive Adversary: Chooses future actions based on previously observed information.
  • Quantum Adversary: Has access to quantum computing resources; relevant for post-quantum cryptography.

Common Cryptographic Attack Models

Security definitions specify the type of attack a scheme should resist. Standard attack models include:

  • Eavesdropping (Passive Attack): Observes messages without interference.
  • Known-Plaintext Attack (KPA): Adversary has access to plaintext-ciphertext pairs.
  • Chosen-Plaintext Attack (CPA): Adversary can request encryptions of arbitrary plaintexts.
  • Adaptive Chosen-Plaintext Attack (Adaptive CPA): Queries depend on previously observed encryptions.
  • Chosen-Ciphertext Attack (CCA): Adversary can request decryptions of arbitrary ciphertexts, excluding the challenge ciphertext.
  • Adaptive Chosen-Ciphertext Attack (CCA2): Can query decryptions both before and after seeing the challenge ciphertext.
  • Chosen-Message Attack (CMA): For digital signatures or MACs, adversary can request signatures for messages of its choice to attempt a forgery.
  • Indistinguishability (IND) Attacks: Adversary tries to distinguish encryptions of two chosen messages, forming the basis for semantic security definitions (IND-CPA, IND-CCA).
  • Key-Compromise Impersonation (KCI): Adversary knows a participant’s secret key and attempts to impersonate other parties to them.
  • Man-in-the-Middle (MITM) Attack: Intercepts and modifies communications between two parties without detection.
  • Replay Attack: Resends valid messages to produce unauthorized effects.
  • Side-Channel Attacks: Exploits physical leakages like timing, power consumption, or electromagnetic emissions.
  • CCA+ (Chosen-Ciphertext with Auxiliary Information): Extends CCA with additional side information or partial key exposure.

Formal Security Considerations

Modern cryptographic security is defined relative to the adversary’s computational limits and interactions:

  • The adversary is assumed to be a PPT algorithm.
  • Success is quantified as the probability of breaking a security property, which must be negligible in the security parameter $n$.
  • Security proofs often reduce breaking the scheme to solving a hard mathematical problem, such as integer factorization, discrete logarithms, or lattice-based problems.
  • Indistinguishability-based definitions (IND-CPA, IND-CCA) formalize semantic security for encryption schemes.
  • Chosen-message security (CMA) formalizes the inability of an adversary to forge valid signatures or MACs, even after querying a signing oracle adaptively.

By explicitly defining adversaries, attack types, and oracles, cryptographers can rigorously prove that a scheme is secure under well-defined assumptions rather than relying on informal intuition.
