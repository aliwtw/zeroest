Introduction to Modern Cryptography

Definition

Modern Cryptography is the rigorous, scientific study of constructing protocols that achieve security goals, such as confidentiality and authenticity, in the presence of adversaries. Unlike classical cryptography, its security is not an assumption but a provable property grounded in computational complexity theory.

Security is formally defined through a game-theoretic framework where the adversary is a probabilistic polynomial-time Turing machine. A cryptographic scheme is considered secure if no such adversary can win a predefined game with a non-negligible advantage. This security is often reduced to the presumed intractability of a well-known computational problem, such as the integer factorization problem (IFP) or the discrete logarithm problem (DLP). This approach allows for the construction of primitives with security guarantees that are mathematically sound and verifiable.

Core Principles of Modern Cryptography

Modern cryptography is built upon three foundational principles that ensure a scientific and rigorous approach to security.

1. Formal Definitions of Security

Cryptographic security is not an intuitive notion; it must be precisely and mathematically defined. This involves a game-theoretic framework where an adversary's capabilities and goals are explicitly modeled. A scheme is declared "secure" only if it can withstand a specific, formalized attack scenario.

Example: Semantic Security for Encryption

An encryption scheme is a triple of algorithms $(\text{Gen}, \text{Enc}, \text{Dec})$, where:

  • $\text{Gen}(1^n)$ is a key generation algorithm that outputs a key $k$.
  • $\text{Enc}_k(m)$ is the encryption algorithm for message $m$.
  • $\text{Dec}_k(c)$ is the decryption algorithm for ciphertext $c$.

Definition: A scheme is semantically secure if, for any two messages $m_0, m_1$ of the same length, no probabilistic polynomial-time (PPT) adversary can distinguish between their encryptions with more than a negligible advantage. Formally, for a security parameter $n$:

$$ \Big| \Pr[A(\text{Enc}_k(m_0)) = 1] - \Pr[A(\text{Enc}_k(m_1)) = 1] \Big| \le \text{negl}(n) $$

where $k \leftarrow \text{Gen}(1^n)$, and $\text{negl}(n)$ is a function that vanishes faster than any inverse polynomial.

2. Precise Assumptions on Computational Hardness

Security is predicated on the existence of computationally "hard" problems. These are problems for which no efficient (i.e., polynomial-time) algorithm is known to exist. If an adversary could solve such a problem efficiently, the cryptographic scheme would be broken.

Examples of Hard Problems

  • Integer Factorization Problem (IFP): Underpins the security of the RSA cryptosystem.
  • Discrete Logarithm Problem (DLP): The foundation for schemes like Diffie–Hellman key exchange and ElGamal encryption.
  • Learning With Errors (LWE): A foundational problem for lattice-based cryptography, a leading candidate for post-quantum schemes.

3. Rigorous Proofs of Security

Security is not just defined or assumed; it is proven. A cryptographic proof is a formal, mathematical argument demonstrating that a scheme is secure, given the stated assumptions. This is often achieved through a reduction.

A security reduction shows that if an adversary could break the cryptographic scheme (e.g., forge a signature), then we could use that adversary as a subroutine to solve the underlying hard problem (e.g., factor an integer). Since we assume the hard problem is intractable, the cryptographic scheme must be secure. This ties the abstract security definition directly to a concrete computational assumption.

Conclusion

Modern cryptography is defined by rigorous mathematical formalism and provable security guarantees. It moves beyond classical ciphers to establish systems secure against well-defined adversaries under precise computational assumptions.
