Signed Binary Representations

Signed vs. Unsigned Numbers

In binary systems, numbers can be interpreted as either unsigned or signed:

  • Unsigned binary represents only non-negative integers.
    An $n$-bit unsigned number ranges from $0$ to $2^n - 1$.
    Example: A 4-bit unsigned number ranges from $0$ to $15$.
  • Signed binary represents both positive and negative integers. Several encoding schemes exist, including:
    • Sign-and-Magnitude
    • 1's Complement
    • 2's Complement (most common)

Sign-and-Magnitude Representation

Definition

In Sign-and-Magnitude representation, the most significant bit (MSB) denotes the sign:

  • 0 for positive numbers
  • 1 for negative numbers

The remaining bits represent the magnitude (absolute value) of the number in binary. For an $n$-bit number:

$$ \text{Bit Pattern: } S \, M_{n-2}M_{n-3}\dots M_0 $$

  • $S$ = sign bit
  • $M_i$ = magnitude bits

Example (4-bit)

Representing $+5$ and $-5$:

Decimal Sign Bit Magnitude (3 bits) Binary
+5 0 101 0101
-5 1 101 1101

Dual Zero Representation

Sign-and-magnitude allows two representations of zero:

  • Positive zero: 0000
  • Negative zero: 1000

This redundancy complicates comparison and arithmetic operations.

Drawbacks

  • Arithmetic Complexity: Addition/subtraction requires separate handling for signs.
  • Inefficient Hardware Implementation: More circuitry is needed to handle sign logic.
  • Redundant Zero: Two representations for zero cause logical ambiguity.

1's Complement Representation

In 1’s complement, negative numbers are represented by flipping all bits of their positive counterparts:

  • MSB indicates sign (0 = positive, 1 = negative)
  • To compute 1’s complement: invert all bits
  • Range for $n$ bits: $-(2^{n-1} - 1)$ to $2^{n-1} - 1$

Example: Represent $-5$ in 4-bit 1’s complement:

  1. 5 in binary: 0101
  2. Flip bits: 1010

Result: $-5$ = 1010

Limitation: Like sign-and-magnitude, 1’s complement has two representations for zero: 0000 and 1111.

2's Complement Representation

2’s complement solves the dual-zero issue and is the standard method used in modern digital systems.

  • MSB indicates sign: 0 = positive, 1 = negative
  • To compute 2’s complement:
    1. Convert positive value to binary
    2. Invert all bits (1’s complement)
    3. Add 1
  • Range for $n$ bits: $-2^{n-1}$ to $2^{n-1} - 1$

Example: Convert $-5$ to 4-bit 2's Complement

  1. 5 in binary: 0101
  2. Invert: 1010
  3. Add 1: 1010 + 0001 = 1011

Result: $-5 = 1011_2$

General Conversion to 2’s Complement

For a decimal number and $n$ bits:

  • If positive: convert to binary and pad to $n$ bits
  • If negative:
    1. Take absolute value and convert to binary
    2. Pad to $n$ bits
    3. Invert all bits
    4. Add 1

Example: Convert $-6$ to 6-bit 2’s complement:

  1. $6 = 000110$
  2. Invert: 111001
  3. Add 1: 111010

Result: $-6 = 111010_2$

Verifying 2’s Complement Values

  • If MSB = 0: read as unsigned
  • If MSB = 1:
    • Invert bits
    • Add 1
    • Convert to decimal and apply minus sign

Example: What is 111010 (6-bit)?

  1. MSB = 1 → negative
  2. Invert: 000101
  3. Add 1: 000110 = 6
  4. Result: -6

Overflow in Binary Arithmetic

Overflow occurs when the result of an arithmetic operation exceeds the range of the fixed bit width.

  • Unsigned: overflow if result > $2^n - 1$
  • Signed (2’s complement):
    • Positive + Positive → Negative
    • Negative + Negative → Positive

Example (4-bit):

$$ 0100\ (4) + 0101\ (5) = 1001\ (-7) \Rightarrow \text{Overflow} $$

Uses and Advantages of Two’s Complement

Where It's Used

  • CPUs & Microcontrollers: Built-in hardware for 2’s complement arithmetic
  • Low-level languages: int, short in C/C++ use 2’s complement
  • Embedded systems & DSP: Sensor data, signals
  • Compilers: Simplifies integer optimizations

Why It’s Preferred

  • Single Zero Representation: Avoids ambiguity
  • Same Logic for Addition/Subtraction: No special hardware needed
  • Consistent Bit Patterns: Simplifies bitwise operations
  • Efficient Modular Arithmetic: Wraps naturally
  • Industry Standard: Supported across all modern architectures

Limitations

  • Asymmetric Range: One extra negative number (e.g., -8 to +7 in 4-bit)
  • Overflow Detection: Requires careful flag handling

Summary

Two’s complement is the industry-standard binary representation for signed integers. Its mathematical elegance, efficient hardware support, and unified treatment of zero and negative numbers make it ideal for modern computing systems.
