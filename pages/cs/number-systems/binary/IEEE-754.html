Binary Floating-Point Representation

Floating-point arithmetic enables computers to represent real numbers—including fractions and extremely large or small values—using a form of scientific notation. In binary systems, floating-point numbers adhere to the IEEE 754 standard, which ensures consistent and accurate numerical computation across platforms.

A normalized binary floating-point number is generally expressed as:

$$ N = (-1)^s \times 1.m \times 2^e $$ Note: Here . represents the binary point.

where:

s = Sign bit (0 for positive, 1 for negative)
m = Mantissa (fractional part after the binary point)
e = Exponent (after applying bias)

The bias allows both positive and negative exponents to be represented as unsigned numbers, simplifying storage and computation.

IEEE 754 Floating-Point Format

Overview

The IEEE 754 standard precisely defines how floating-point numbers are represented and manipulated in binary. It specifies the encoding of real numbers using sign, exponent, and mantissa fields, along with rules for rounding, special values (such as infinity and NaN), and numerical stability.

Structure of IEEE 754 (Single Precision)

The 32-bit floating-point format consists of three main components:

Sign Bit (1 bit) → 0 for positive, 1 for negative
Exponent (8 bits) → Stored with a bias of 127
Mantissa (23 bits) → Binary digits following the binary point (with an implicit leading 1)

The floating-point value is reconstructed by the formula:

$$ (-1)^s \times 1.m \times 2^{E - 127} $$

Conversion Steps (Decimal → IEEE 754)

To convert a decimal number into IEEE 754 format, follow these steps:

1. Convert the decimal number to binary, separating integer and fractional parts.
2. Normalize the binary number to the form \(1.xxxxx \times 2^e\).
3. Determine the sign bit: 0 if positive, 1 if negative.
4. Calculate the biased exponent: \(e + 127\).
5. Extract the mantissa bits (the 23 bits following the binary point).
6. Assemble the 32-bit IEEE 754 value as: [sign][exponent][mantissa].

Example: Converting 5.75 to IEEE 754 (32-bit)

Step 1: Convert to binary

$$ 5 = 101_2,\quad 0.75 = 0.11_2 \Rightarrow 5.75 = 101.11_2 $$

Step 2: Normalize

$$ 101.11_2 = 1.0111_2 \times 2^2 $$

Step 3: Determine sign bit

$$ \text{Positive} \Rightarrow s = 0 $$

Step 4: Compute biased exponent

$$ e = 2,\quad E = 2 + 127 = 129,\quad E_{bin} = 10000001_2 $$

Step 5: Extract mantissa

$$ \text{Mantissa (omit leading 1)} = 01110000000000000000000 $$

Step 6: Assemble IEEE 754 representation

Sign = 0
Exponent = 10000001
Mantissa = 01110000000000000000000

Final IEEE 754 (binary):

0 10000001 01110000000000000000000

In hexadecimal:

$$ \text{IEEE 754 (hex)} = \texttt{0x40B80000} $$

Special Values

Exponent = 255 (all 1s), Mantissa = 0 → ±Infinity
Exponent = 255, Mantissa ≠ 0 → NaN (Not a Number)
Exponent = 0, Mantissa ≠ 0 → Denormalized number
Exponent = 0, Mantissa = 0 → Zero (positive or negative)

Summary

- IEEE 754 employs normalized scientific binary notation with an implicit leading 1.
- Exponent biasing enables unsigned storage of both positive and negative exponents.
- The 23-bit mantissa limits precision, affecting exactness for some decimal values.
- Special encodings are used to represent infinity, NaN, and subnormal numbers.
